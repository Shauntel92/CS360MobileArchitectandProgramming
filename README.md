# CS360MobileArchitectandProgramming
**Author: Shauntel Hall**

This project is a fully functional Android mobile application developed as part of a full development cycle, from initial planning and UI design to implementation and testing. The goal of the app is to allow users to track their daily weight, set a goal weight, and receive notifications when that goal is reached. The app was designed to address a simple but important user need, to provide an easy, organized way to monitor weight progress over time. Many existing apps are overloaded with features, so this project focused on creating a clean, user-friendly experience centered around core functionality.

To support user needs, the application includes the following key screens and features:
Login/Create Account Screen
Allows users to securely log in or create a new account
Designed to be simple and intuitive with clear input fields and actions
Weight Log Screen
Displays daily weight entries in a structured list (grid-style layout)
Users can add, update, and delete entries (CRUD functionality)
Includes clear labels and organized layout for easy readability
Goal Weight & Notifications
Users can set a goal weight
The app checks progress and triggers an SMS notification when the goal is reached if permission is granted.
The UI was designed with the user in mind by keeping layouts clean, minimizing unnecessary steps, and organizing features in a logical flow. Navigation between screens follows a natural progression, which helps reduce confusion and improve usability. Overall, the design was successful because it prioritizes simplicity and clarity.

I approached development in stages, starting with the UI design and then gradually building functionality on top of it. I focused on getting the core features working first, such as navigation between screens, before implementing more complex functionality like database operations and SMS notifications. Testing was an important part of the process because it helped identify issues early, especially with navigation, data storage, and permissions. It also revealed how small errors in logic or UI connections could impact the overall user experience.

One of the main challenges I faced was integrating multiple components, such as connecting the UI to the database and ensuring data was correctly stored and displayed. Troubleshooting these issues required patience and problem-solving, especially when errors were not immediately clear. I had to think creatively to debug problems, test different solutions, and adjust my approach when something wasn’t working. This process helped me become more confident in handling unexpected issues and finding solutions independently.

One area where I feel I was particularly successful was implementing the database and CRUD functionality. Being able to store user data, display it dynamically, and allow users to update and delete entries demonstrated my understanding of both backend logic and front-end interaction. This feature ties together multiple aspects of development, including data management, UI updates, and user interaction, making it a strong example of the skills I developed throughout this project.

This project helped me better understand the full lifecycle of mobile app development. It strengthened my ability to think from a user’s perspective, design intuitive interfaces, and build functional applications step by step. It also gave me more confidence in troubleshooting and working through challenges independently.
